## 项目概述
开发一个2D上帝视角多人战斗肉鸽游戏，项目名称为"战场幸存者"

## 技术栈
- 前端：HTML5 + Phaser.js
- 后端：Node.js + WebSocket
- 版本控制：Git + GitHub

## 项目结构
```
battle-survivors/
├── client/
│   ├── assets/
│   │   ├── images/

│   │   └── sprites/
│   ├── js/
│   │   ├── scenes/
```

## package.json配置
- 项目名称：battle-survivors
- 项目描述：2D上帝视角多人战斗肉鸽游戏
- 主入口文件：server/server.js
- 脚本命令：
  - `"start": "node server/server.js"`
  - `"dev": "nodemon server/server.js"`

  - express: ^4.18.2

- 开发依赖：
  - nodemon: ^3.0.1

## 第二阶段：前端基础开发

### 指令4：创建前端入口文件
创建client/index.html文件，包含：
1. 基本HTML结构
2. 引入Phaser.js库
3. 创建游戏画布
4. 添加基本的CSS样式

要求：
- 画布大小为800x600像素
- 居中显示
- 深色背景
- 响应式设计

### 指令5：实现基础游戏逻辑
创建client/js/game.js文件，实现以下功能：
1. 初始化Phaser游戏实例
2. 创建游戏场景
3. 实现玩家角色（可移动、可射击）
4. 实现基本的敌人AI
5. 实现碰撞检测系统
6. 实现简单的UI界面

要求：
- 使用简约的像素风格
- 玩家使用方向键移动，空格键射击
- 敌人随机生成并向玩家移动
- 子弹击中敌人后双方都消失
- 显示玩家血量和分数

### 指令6：添加游戏资源
创建基本的游戏资源：
1. player.png: 16x16像素的玩家角色
2. enemy.png: 12x12像素的敌人角色
3. bullet.png: 4x4像素的子弹
4. background.png: 800x600像素的背景图

要求：
- 使用简约的像素风格
- 颜色鲜明，易于识别

## 第三阶段：后端开发

### 指令7：创建WebSocket服务器
创建server/server.js文件，实现以下功能：
1. 创建Express服务器
2. 集成WebSocket服务器
3. 实现房间管理系统
4. 实现玩家连接管理
5. 实现游戏状态同步

要求：
- 支持创建和加入房间
- 支持2-4人同时游戏
- 实时同步玩家位置和状态
- 处理玩家断开连接

### 指令8：实现游戏逻辑
创建server/gameLogic.js文件，实现以下游戏逻辑：
1. 房间状态管理
2. 玩家状态同步
3. 敌人生成逻辑
4. 碰撞检测验证
5. 游戏结束判定

要求：
- 服务器权威验证，防止作弊
- 定时生成敌人
- 处理玩家死亡和复活
- 计算游戏分数

## 第四阶段：肉鸽功能实现

### 指令9：随机地图生成
实现随机地图生成系统：
1. 创建client/js/utils/mapGenerator.js
2. 实现随机地图算法
3. 生成不同的地形和掩体
4. 确保地图每次都不同

要求：
- 使用瓦片地图系统
- 包含墙壁、掩体、空地等地形
- 确保玩家有足够的移动空间
- 地图边缘有边界

### 指令10：道具系统
实现道具系统：
1. 创建client/js/objects/Item.js
2. 实现道具生成和掉落
3. 实现道具拾取和使用
4. 添加不同类型的道具

道具类型包括：
- 血量恢复
- 武器升级
- 速度提升
- 伤害增强
- 范围攻击

### 指令11：升级系统
实现角色升级系统：
1. 创建client/js/objects/Upgrade.js
2. 实现升级选择界面
3. 实现不同的升级效果
4. 处理升级逻辑

升级选项包括：
- 血量上限提升
- 攻击力提升
- 射击速度提升
- 移动速度提升
- 暴击率提升
- 范围伤害
